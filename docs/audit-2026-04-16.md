## Anti-Patterns Verdict

Mostly clean, but not fully out of the woods. The design avoids the worst AI tells — no gradient text, no glassmorphism overuse, no cyan-on-dark, no neon accents, no hero metric templates. But:

1. **Confirmed banned pattern** — `About.astro:58` had `border-l-2 border-forest/30`, a left-border accent stripe on a callout block. **Fixed.**
2. **Top border stripe** — `Services.astro:57` had `border-t-2 border-t-forest` on every service card. **Fixed.**
3. **Reflex font choices** — Instrument Serif + Plus Jakarta Sans. Both are on the overused/AI-reflex list. They work fine individually but together they signal AI-assembled design.
4. **Nested card pattern** — `FewerClients.astro:54-55` placed `bg-white/[0.04] border rounded-xl` cards inside a `bg-charcoal rounded-2xl` container. **Fixed.**

## Detailed Findings by Severity

### P1 — Major

**[P1] Button toggle groups lack semantic grouping** ✅ Fixed
- **Location:** `src/pages/contact/index.astro:96–123` (Project Type), `:127–153` (Scope)
- **Category:** Accessibility
- **Impact:** Screen readers have no way to announce these as a group. A user navigating with VoiceOver hears three isolated buttons with no context that they're mutually exclusive choices.
- **WCAG:** 1.3.1 Info and Relationships
- **Fix applied:** Changed `<label>` to `<p id="...">`, added `role="radiogroup"` + `aria-labelledby` to container, added `role="radio"` + `:aria-checked` to each button.

---

**[P1] Required form fields missing `aria-required` and error associations** ✅ Fixed
- **Location:** `src/pages/contact/index.astro:64` (Name), `:82` (Email), `:208` (Message)
- **Category:** Accessibility
- **Impact:** Screen reader users don't know fields are required until submission fails. Error messages existed but weren't programmatically associated with their inputs.
- **WCAG:** 3.3.1 Error Identification, 3.3.2 Labels or Instructions
- **Fix applied:** Added `aria-required="true"` and `aria-describedby="field-error"` to each required input; added matching `id` to each error `<p>`.

---

**[P1] SVG icons in footer links missing `aria-hidden`** ✅ Fixed
- **Location:** `src/components/Footer.astro:17–19` (LinkedIn), `:28–30` (GitHub)
- **Category:** Accessibility
- **Impact:** Screen readers announce SVG path data or generic "image" alongside the link text, creating garbled output. Parent `<a>` elements already have `aria-label`.
- **WCAG:** 1.1.1 Non-text Content
- **Fix applied:** Added `aria-hidden="true"` to both SVGs.

---

**[P1] Availability.astro SVGs missing `aria-hidden`** ✅ Fixed
- **Location:** `src/components/Availability.astro:28–30` (checkmark), `:69–71` (X icon), lines 36/42/48/54/77/83/89 (bullet dots)
- **Category:** Accessibility
- **Impact:** Decorative checkmarks, X icons, and bullet dots announced as unnamed images by screen readers.
- **WCAG:** 1.1.1 Non-text Content
- **Fix applied:** Added `aria-hidden="true"` to all decorative SVGs.

---

**[P1] Mobile menu button touch target below 44px** ✅ Fixed
- **Location:** `src/components/Nav.astro:36`
- **Category:** Accessibility / Responsive
- **Impact:** Button used `p-2` (8px padding) giving a ~40×40px target. WCAG 2.5.5 requires 44×44px minimum.
- **WCAG:** 2.5.5 Target Size
- **Fix applied:** Changed `p-2` to `p-3` (48×48px target). Also added `aria-hidden="true"` to the hamburger SVG.

---

**[P1] `border-l-2` accent stripe — BANNED PATTERN** ✅ Fixed
- **Location:** `src/components/About.astro:58`
- **Category:** Anti-Pattern
- **Impact:** One of the most recognizable AI design tells. Made the working style callout look templated.
- **Fix applied:** Replaced with `bg-forest-pale rounded-lg px-5 py-4`; text changed from `text-gray-500` to `text-forest/75` (gray on a tinted surface violates the design rule).

---

**[P1] `border-t-2` accent stripe on service cards — BANNED PATTERN** ✅ Fixed
- **Location:** `src/components/Services.astro:57`
- **Category:** Anti-Pattern
- **Impact:** Top-edge colored stripe on every card — same templated feel as the left stripe. Applied to all 5 service cards.
- **Fix applied:** Removed `border-t-2 border-t-forest` entirely. Cards rely on content, typography, and badge chips to carry visual weight.

### P2 — Minor

**[P2] Contact form button grids not responsive** ✅ Fixed
- **Location:** `src/pages/contact/index.astro:98` (`grid-cols-3`), `:129` (`grid-cols-2`)
- **Category:** Responsive Design
- **Impact:** At 320–375px viewport, three buttons in a fixed 3-column grid collapse to ~80px each — text wraps awkwardly.
- **Fix applied:** Changed to `grid-cols-1 sm:grid-cols-3` and `grid-cols-1 sm:grid-cols-2`.

---

**[P2] Nested card pattern in FewerClients** ✅ Fixed
- **Location:** `src/components/FewerClients.astro:20` (outer), `:54–55` (inner cards)
- **Category:** Anti-Pattern
- **Impact:** `bg-charcoal rounded-2xl` container held `bg-white/[0.04] border-white/[0.08] rounded-xl` inner cards — double card nesting that reads as AI template.
- **Fix applied:** Replaced inner card boxes with `pt-6 border-t border-white/10` dividers. Flatter, more considered.

---

**[P2] Logo images inconsistently optimized** ⚠️ Open
- **Location:** `public/images/logos/`, referenced in `src/components/Work.astro`
- **Category:** Performance
- **Impact:** 5 of 8 client logos are PNG/SVG; only 2 use AVIF. Inconsistent format strategy adds unnecessary bytes.
- **Recommendation:** Convert PNG logos to AVIF/WebP using Astro's `<Image>` component or a build step. Use SVG where logos are flat vector.

---

**[P2] Missing `width`/`height` on images (CLS risk)** ⚠️ Open
- **Location:** `src/components/About.astro:69–74` (Shopify logo), `src/components/Work.astro:132–149` (screenshots and project logos)
- **Category:** Performance
- **Impact:** Without intrinsic dimensions, browser can't reserve space before image loads → Cumulative Layout Shift. Note: images inside `aspect-*` wrappers are largely protected; the Shopify logo (`h-8` only, no width) is the main risk.
- **Recommendation:** Add explicit `width` and `height` to the Shopify logo image; use Astro's `<Image>` component for Work screenshots.

---

**[P2] Both fonts are AI-reflex list entries** ⚠️ Open
- **Location:** `src/layouts/BaseLayout.astro:42`, `tailwind.config.mjs`
- **Category:** Anti-Pattern
- **Impact:** Instrument Serif and Plus Jakarta Sans are both on the overused/AI-reflex list. Used together they signal AI-assembled design to anyone who knows what to look for.
- **Recommendation:** Revisit at next design iteration via `/typeset` with the brand context now established in `.impeccable.md`.

---

**[P2] Nav backdrop-blur is mild glassmorphism** ⚠️ Open
- **Location:** `src/components/Nav.astro:9` — `backdrop-blur-sm bg-white/95`
- **Category:** Anti-Pattern
- **Impact:** Functionally justified for a sticky nav, but combined with other tells it contributes to the AI aesthetic fingerprint.
- **Recommendation:** Could swap to a solid `bg-white border-b` — reads as more intentional. Low priority.

---

**[P2] Raw Tailwind gray scale used without token abstraction** ⚠️ Open
- **Location:** All components — `text-gray-400/500/600`, `border-gray-200/300`, `bg-white` appear in every `.astro` file
- **Category:** Theming
- **Impact:** No neutral token layer means color changes require touching every component. If a dark mode or rebrand is ever needed, this requires file-by-file edits.
- **Recommendation:** Add neutral tokens to `tailwind.config.mjs` (`surface`, `muted`, `subtle`) and replace raw `gray-*` usage systematically. Low urgency for a small site.

---

**[P2] Error color (`text-red-600`) outside design token system** ⚠️ Open
- **Location:** `src/pages/contact/index.astro:74, 91, 218`
- **Category:** Theming
- **Impact:** Error state uses default Tailwind red rather than a defined design token.
- **Recommendation:** Add an `error` color to `tailwind.config.mjs`, use `text-error` instead.

## Patterns & Systemic Issues

- **SVG `aria-hidden` missing throughout** — Fixed across Nav, Footer, Availability. Still worth checking ProcessTimeline.astro (SVGs injected via `set:html`) and Work.astro.
- **Gray scale used without token abstraction** — All 12 components use raw `gray-*` classes directly. One refactor pass would close this gap.
- **Contact form was the highest-risk page** — Contained the most P1 issues. Fully addressed in this audit cycle.

---

## Positive Findings

- Fathom and sal.js are both `defer`-loaded — no render blocking
- `display=swap` correctly set in the Google Fonts URL — no invisible text flash
- `preconnect` to both `fonts.googleapis.com` and `fonts.gstatic.com`
- Mobile menu has `aria-expanded` correctly toggled
- Form errors use `role="alert"` — screen reader users get real-time validation feedback
- `font-smoothing: antialiased` applied globally
- `scroll-behavior: smooth` set globally
- Animations use `opacity` and `transform` via sal.js — no layout-thrashing properties
- `once: true` on sal.js — elements don't re-animate on scroll-back
- Hero and CTA sections use high-contrast backgrounds (charcoal/forest) with cream text
