  Overall Impression

  Strong bones, high design quality. The main gap is contrast — ui-faint (#9ca3af) is used as readable text throughout the site and fails WCAG AA everywhere it appears on white or cream.
  It's essentially a decorative-only color being used for content. Fix that one systemic token issue and the accessibility score jumps.

  What's Working

  1. Focus management is exemplary. Every interactive element uses focus-visible: (not focus:), with consistent ring-forest indicators. The contact form has aria-required,
  aria-describedby, and role="alert" on errors — genuinely good work.
  2. Animation system is clean. prefers-reduced-motion respected at every layer (sal.js, hero word reveal, footer wordmark, work screenshot clip, nav underline, check-draw).
  Transform/opacity only throughout.
  3. Token discipline. Semantic UI token names (ui-border, ui-faint, ui-muted, ui-body) are used consistently everywhere. Zero raw hex colors found in any component.

  Priority Issues

  [P1] ui-faint used as readable text — systemic WCAG AA failure

  - Location: Services.astro (type labels), Work.astro (type labels), Testimonials.astro (cite attribution), ClientLogos.astro (eyebrow label), Footer.astro (copyright)
  - Category: Accessibility
  - Impact: ui-faint (#9ca3af) on white = 2.54:1. On cream = 2.24:1. Both fail WCAG AA (4.5:1 required for normal text). Affects every page.
  - WCAG: 1.4.3 Contrast (Minimum), Level AA
  - Fix: Replace text-ui-faint with text-ui-muted (#6b7280, 4.83:1 on white) for all non-decorative text. Reserve text-ui-faint for decorative or non-text UI only.
  - Suggested command: /polish

  [P1] white/60 on forest background fails contrast

  - Location: FooterCTA.astro line 16 (text-white/60 on bg-forest)
  - Category: Accessibility
  - Impact: white/60 on forest = 3.81:1. The subtitle "Have a Shopify project in mind? Let's talk about it." fails WCAG AA.
  - WCAG: 1.4.3 Contrast (Minimum), Level AA
  - Fix: Increase to text-white/80 (≈6.0:1) or text-white/75.
  - Suggested command: /polish

  [P1] forest/75 on forest-pale background fails contrast

  - Location: About.astro line 63 (working style callout: text-forest/75 on bg-forest-pale)
  - Category: Accessibility
  - Impact: 3.67:1 — fails WCAG AA for the callout body text.
  - WCAG: 1.4.3 Contrast (Minimum), Level AA
  - Fix: Use text-forest (full opacity, ~7.2:1 on forest-pale) or text-forest/90.
  - Suggested command: /polish

  [P1] No skip navigation link

  - Location: BaseLayout.astro / Nav.astro — <main id="content"> exists but no skip link
  - Category: Accessibility
  - Impact: Keyboard users must tab through all 5 nav links + CTA on every page load before reaching content. Critical for screen reader and power keyboard users.
  - WCAG: 2.4.1 Bypass Blocks, Level A
  - Fix: Add a visually-hidden skip link before the nav: <a href="#content" class="sr-only focus:not-sr-only ...">Skip to content</a>
  - Suggested command: /harden

  [P2] Founder photo missing width/height — CLS risk

  - Location: About.astro line 16-22 (/images/david-lindahl.jpg)
  - Category: Performance
  - Impact: Without dimensions, the browser can't reserve space before the image loads, causing layout shift (CLS). The photo is inside an aspect-[3/4] container which mitigates this
  somewhat, but explicit dimensions are still best practice.
  - Fix: Add width="280" height="373" (or actual photo dimensions). The container handles visual sizing.
  - Suggested command: /optimize

  [P2] Nav "Let's Chat" CTA below 44px touch target

  - Location: Nav.astro lines 27-31 and 59-63 (py-2.5 text-sm)
  - Category: Responsive Design
  - Impact: py-2.5 (10px) + text-sm (14px at ~1.5 line-height = ~21px) = ~41px total height. Falls short of the 44px WCAG minimum touch target for mobile.
  - WCAG: 2.5.5 Target Size, Level AAA (advisory); iOS HIG requires 44pt minimum
  - Fix: Change py-2.5 to py-3 (12px padding, ~45px total).
  - Suggested command: /polish

  [P2] 7 microlink.io screenshot requests on page load

  - Location: Work.astro line 116 (the screenshotSrc calculation)
  - Category: Performance
  - Impact: 5 projects with live URLs each trigger a third-party API call. Even with loading="lazy", these requests fire as rows enter the viewport. If microlink is slow or rate-limits,
  the hover reveal shows nothing (fallback handles it, but it's visible latency).
  - Fix: Pre-generate screenshots at build time (via a simple script using microlink's download option) and commit them as static images. Eliminates runtime dependency.
  - Suggested command: /optimize

  [P3] 15 Google Font files across 2 families

  - Location: BaseLayout.astro line 42
  - Category: Performance
  - Impact: Spectral loads 4 styles; Jost loads 8 styles (300, 400, 500, 600, 700 + italic variants). Many of these weights may not all be used. Each is a separate HTTP request + download.
  - Fix: Audit which Jost weights are actually used in Tailwind classes. font-light (300) and font-bold (700) are likely unused — removing them saves ~2 network requests and payload.
  - Suggested command: /optimize

  ---
  Patterns & Systemic Issues

  1. ui-faint is misassigned as a text color — it's named and calibrated as a decorative/icon color (gray-400 = 2.54:1 on white), but it's used for readable text in 5+ components. The
  token itself is fine for borders and icons; the systemic error is using it for text content. Every instance needs swapping to ui-muted.
  2. Semi-transparent text on colored backgrounds consistently fails — white/60 on forest and forest/75 on forest-pale both fail. When text is placed on brand-colored surfaces, use fully
  opaque alternatives rather than alpha-reduced brand colors.
